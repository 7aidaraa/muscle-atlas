# الأطلس العضلي — حالة المشروع للانتقال إلى Claude Code

## ما هو المشروع
منصة تعليمية رياضية عربية بالكامل: خريطة جسد تفاعلية + ملفات عضلات + وضع كابتن لتسجيل التمارين + حاسبة أيض + سجل محفوظ. ملف واحد قائم بذاته: `الأطلس-العضلي.html` (~١١١٨ سطراً، HTML+CSS+JS خام بلا أطر).

## البنية المعمارية
- **نافذة دخول (gate):** حاسبة Mifflin-St Jeor — تُدخل العمر/الوزن/الطول/النشاط/الخبرة/الهدف، تحسب BMR وTDEE وBMI، تصنّف المستخدم، ثم `enterApp()`.
- **خريطة ثابتة فوق:** جسدان (أمامي/خلفي) جنباً إلى جنب، لا يزولان. مسارات SVG من مكتبة body-highlighter (رخصة MIT). الدالة `renderBodyMap()`.
- **لوحة محتوى متبدّلة تحت (`#panel`):** تتنقّل بين: ترحيب ← بطاقة عضلة ← قائمة تمارين ← كابتن ← سجل. التنقّل عبر `nav[]` + `pushNav()`/`goBack()` + `renderPanel()`. لا نظام شاشات منفصلة (أُلغي).

## الميزات المنجزة
1. حاسبة أيض علمية موثّقة (Mifflin-St Jeor + معاملات نشاط قياسية).
2. خريطة تفاعلية: نقر العضلة يفتح ملفها، إضاءة لحظية + صوت.
3. ملف كل عضلة (١٥ عضلة): اسم عربي/لاتيني، وظيفة، استشفاء خاص بساعاته، عضلات مساندة.
4. تمارين متدرّجة (تأسيس→صعب) مع توصية حسب المستوى، وجرعة حسب الهدف (ACSM 2026).
5. عرض الحركة: صورتان متبدّلتان من free-exercise-db (Unlicense)، واحتياطي رسم متحرك.
6. وضع الكابتن: خطوات مرقّمة + خطأ شائع + كلمة كابتن (تحفيز/تحذير سلامة/تنبيه استشفاء) + إدخال التكرارات الفعلية بعدّاد −/+ وتسجيل كل مجموعة.
7. السجل: ما أُنجز بتكراراته، حالة الاستشفاء الحيّة، توصية الجلسة القادمة، زرّ مسح.
8. الحفظ: localStorage (محلّي على الجهاز) + زرّ حفظ يدوي + إشعار + مؤشّر وقت.
9. تلوين العضلة المُنجزة أخضر حتى انتهاء استشفائها (يعود تلقائياً).
10. وضع "ابنِ تمريني اليوم" (`buildToday`).
11. العضلات المتضادّة (`ANTAGONIST`).
12. تصميم متجاوب (media queries لكل الأحجام).

## نقاط تحتاج اختباراً/انتباهاً (لم تُعايَن بصرياً)
- **التلوين التفاعلي:** عولج بخفض تباين لون العضلة الافتراضي (#1e2730) قرب الظلّ (#161d26). يحتاج تأكيداً أن العضلات تبدو خافتة وتُضيء عند التفاعل فقط.
- **localStorage في artifacts:** يعمل في الملف المحلّي، لكنه قد يُمنع في بعض السياقات (محاط بـtry/catch).
- **الصور من GitHub:** تحتاج اتصالاً بالإنترنت؛ يوجد احتياطي رسم متحرك عند الفشل.
- الربط بين تمارين المنصة وصور free-exercise-db تقريبي (أقرب مكافئ بصري) لا حرفي.

## ملاحظات للتطوير في Claude Code
- المشروع ملف واحد؛ يُنصح بتقسيمه (HTML/CSS/JS منفصلة) لسهولة الصيانة.
- شغّل الملف في متصفّح وعايِن فعلياً — أكثر الأخطاء السابقة كانت بصرية لم تُكتشف إلا بصور المستخدم.
- القاعدة اللغوية الملزِمة: الواجهة كلها عربية فصحى، لا حرف لاتيني ظاهر للمستخدم (المصطلح الأجنبي يُكتب صوتياً بالعربية).
- الأرقام تُعرض بالأرقام العربية (دالة `ar()`).

## أولويات مقترحة للمرحلة القادمة
1. تأكيد إصلاح التلوين بصرياً (الأهم).
2. اختبار دورة كاملة: دخول → اختيار عضلة → تمرين → تسجيل تكرارات → سجل → حفظ → إعادة فتح.
3. تقسيم الملف لو طال التطوير.
4. (اختياري) مزامنة سحابية بدل localStorage لربط الأجهزة.
